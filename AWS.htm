DynamoDBへのGet,Postの処理

例）

import json

import os

import boto3

 

REGION = os.environ['REGION']

TABLE = os.environ['TABLE']

 

dynamo = boto3.resource('dynamodb', region_name=REGION)

table = dynamo.Table(TABLE)

 

 

def lambda_handler(event, context):

 

    if event['httpMethod'] == 'GET':

        data = table.scan()['Items']

    elif event['httpMethod'] == 'POST':

        postData = json.loads(event['body'])

        table.put_item(Item = postData)

        data = {'message': 'successfull'}

   

    return {

        'statusCode': 200,

        'body': json.dumps(data)

    }

AWS上にデータウェハウスを構築することのできるマネージド型サービス

 

特徴

高速でスケーラブルな費用対効果の高いマネージド型のDWH

※DWH=データウェアハウス　直訳"データの倉庫"

→定義：意思決定のため、目的別に編成され、統合された時系列で、削除や更新しないデータの集合体

→イメージ：データをそのまま突っ込む・データを使う側が加工する

ノードタイプ　→　利用するデータサイズやケースに応じてノードタイプを選択

RA3：もっとも高性能→データ量の増大が予想される場合に推奨

DS2：高密度ストレージ→低コスト・未圧縮で１TB以下で推奨

DC：高密度コンピューティング→高スループット、低レイテンシ用・データが500GBを超える場合に最適

Streamsはリアルタイムにデータを処理して、

StreamsとSQSとの違い

Streamsはデータ、SQSはメッセージ

セキュリティまとめ

2023年9月5日

17:19

・セキュリティグループ

用途）インスタンス等、個々の通信制限

注意）特定の送信元からの通信を遮断できない

→ルールは許可する通信しか指定できない

 

・ネットワークACL

用途）サブネットレベル（複数インスタンス）への通信制限

注意）ステートレスの為、戻りパケットの許可が必要

 

・Network Firewall

用途）複数のサブネットに対して、一元的に通信を制限

特定のプロトコル（ポートに頼らない）やドメインに対して制限が可能

注意）通信経路（往復分）にFirewallを経由させるようにルート登録が必要

ステートレス→ステートフルでルールを評価する

 

・WAF

用途）Webアプリケーションを保護

SQLインジェクションやDOS攻撃に対して

注意）適用できるのは、ALB・CloudFront・API Gatewayのみ（学習済みなのは）
